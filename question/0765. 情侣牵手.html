题目

N 对情侣坐在连续排列的 2N 个座位上，想要牵到对方的手。 计算最少交换座位的次数，以便每对情侣可以并肩坐在一起。 一次交换可选择任意两人，让他们站起来交换座位。
人和座位用 0 到 2N-1 的整数表示，情侣们按顺序编号，第一对是 (0, 1)，第二对是 (2, 3)，以此类推，最后一对是 (2N-2, 2N-1)。
这些情侣的初始座位  row[i] 是由最初始坐在第 i 个座位上的人决定的。

示例

示例 1:
输入: row = [0, 2, 1, 3]
输出: 1
解释: 我们只需要交换row[1]和row[2]的位置即可。

示例 2:
输入: row = [3, 2, 0, 1]
输出: 0
解释: 无需交换座位，所有的情侣都已经可以手牵手了。

说明

  • len(row) 是偶数且数值在 [4, 60]范围内。
  • 可以保证row 是序列 0...len(row)-1 的一个全排列。

题解

首先我们了解几个概念
  • 所有的数字都整除2，因为我们发现整除2之后，如果是情侣，整除2之后的结果是相等的
  • 最后所有的情侣都是由索引 n,n+1 两个是情侣，并且 n 是偶数，因为 n 如果不是偶数，会把点孤立起来，两边是两个情侣，无法满足要求
我们使用将原始序列转换成两个两个的组合[x0,y0][x1,y1],[x2,y2]...[xn,yn]，[x,y]说明 x 和 y 建立一条边，成为一个集合
可以得出，在一个集合中交换可以使次数最少，因为如果不在一个集合，我们第一次交换，会把两个集合转换成一个集合，变成同一个集合的问题，这样会多出一步
每个集合会是下面几种情况之一
  • [x,x]：这种直接满足条件，并且集合会封闭，因为没有新的元素和外面的集合建立联系。交换次数为0，集合元素个数是1，对数是1
  • [x,y][y,x]：这种需要交换一次，并且集合会封闭，因为没有新的元素和外面的集合建立联系。集合元素数量是2，对数是2
  • [x1,y1][y1,x2]...[xn,x1]：剩下的都是这种情况，我们可以这样推断，集合想要和外面的元素建立联系需要一个在这个集合中出现一次的元素，因为题目中说肯定满足，那整个集合中肯定每个元素出现两次（一对情侣）。并且每个集合中肯定有一个元素师集合中的（建立联系），另一个元素如果不是集合中的，说明集合还没有封闭，两一个如果是集合中的，那么集合就封闭了，因为没有新的元素和外面的元素建立联系。集合元素 n 个，集合对数 n。我们通过观察发现需要交换 n-1 次是集合满足条件，因为整个集合组成环状，我们沿着环交换就可以
我们观察发现不管哪一种形式，交换次数等于 n-1。因此我们需要遍历整个集合，将元素划分到集合，计算每个集合的交换次数，求和就是最终的交换次数
我们可以使用并查集，也可以不使用并查集，而是深度搜索，因为这里的并查集很特殊，所有的并查集中的元素都构成一个环（n 个对，n-1条边）每个对的父节点是唯一的，每个父节点的孩子节点也是唯一的。因此我们只需要从一个对开始遍历，标记它已经被添加到集合中，然后找到它链接的下一个对，继续遍历，直到找到的对已经被添加到集合（闭环）。然后我们通过遍历的对数量-1获取交换次数
